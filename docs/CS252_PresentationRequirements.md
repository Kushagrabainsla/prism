# CS252 Presentation Requirements

**Due:** Monday by 11:59pm  
**Points:** 10

Project presentations will be 5 minutes, with 2 additional minutes for questions. (If you run too long or too short, you will lose points).

For tips on making a good presentation, review ["How to Avoid Death by PowerPoint"](https://www.google.com/search?q=How+to+Avoid+Death+by+PowerPoint).

## Rubric

| Criteria | Ratings | Points |
|----------|---------|--------|
| **Content of presentation** | Interesting PL ideas presented to class | 3 pts |
| | Shallow or cursory discussion only | 2 pts |
| | Minimal interesting content | 1 pts |
| | Presentation not done | 0 pts |
| **Clarity of presentation** | Ideas are clear and easy to follow | 3 pts |
| | Mostly clear, some confusing parts of the talk | 2 pts |
| | Confusing, hard to follow presentation | 1 pts |
| | Presentation not done | 0 pts |
| **Style** | Interesting, engaging presentation<br>Good motivation, strong visuals (where applicable), informative examples. Not too much text. | 2 pts |
| | Professional presentation<br>Easy to follow for those interested in the topic, but lacking in motive or enthusiasm to catch the interest of other listeners. | 1 pts |
| | Wall of text<br>The slides are a wall of text, which the presenter just reads to the audience. | 0.5 pts |
| | Presentation not done | 0 pts |
| **Time management** | Presentation fits time slot | 2 pts |
| | Presentation ran long<br>Either the talk seems rushed, or the talk ran over time. | 1 pts |
| | Presentation shorter than allocated time<br>Finishing a minute early is fine. Finishing 3 minutes early (in a talk this short) is not. | 1 pts |
| | Presentation not done | 0 pts |

**Total Points:** 10
